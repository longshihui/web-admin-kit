# 文档规范

本文件用于约束通用文档写作和维护。具体文档目录、站点构建和发布流程以项目级规范为准。

## 文档原则

- 文档应面向使用者解释能力、边界和正确用法，而不是复述源码实现。
- 公共 API、默认行为、错误行为或使用方式变化时，应同步更新文档。
- 示例应可运行或接近真实使用场景，避免只有伪代码。
- 文档中的术语、参数名、事件名和代码导出应与实现保持一致。

## README

- README 应说明能力定位、适用场景、基础用法和最小示例。
- README 不应堆叠所有实现细节；复杂 API、设计取舍和边界应拆到独立文档。
- README 中的示例应优先展示推荐用法，而不是边缘能力。

## API 文档

- API 文档应说明参数、返回值、默认值、错误行为和边界条件。
- 类型复杂时，应解释关键字段语义，而不是只贴类型定义。
- 异步 API 应说明触发时机、完成时机、错误路径和并发行为。
- 组件 API 应说明 props、emits、slots、暴露方法和 `v-model` 更新时机。

## 示例与迁移

- 示例代码应使用清晰命名，避免无意义变量和与真实场景脱节的数据。
- 存在多种用法时，应先展示推荐路径，再说明替代路径。
- 发生破坏性变更或迁移成本较高时，应提供迁移说明。
- 文档不要承诺尚未实现或没有测试覆盖的能力。

## 维护要求

- 修改代码时发现文档过期，应在当前影响范围内同步修正。
- 删除能力时，应删除或更新对应文档示例。
- 文档变更后，应按项目级规范运行文档检查或构建命令。

# 代码重构

本文件用于约束重构行为。重构必须服务当前任务，不应扩大到无关模块。

## 重构边界

- 只重构当前需求直接影响的代码路径。
- 不为了让代码“更优雅”而改动无关模块。
- 重构应优先保持行为不变，再整理结构、命名和边界。
- 如果重构会影响公共 API、兼容性、数据结构或调用方式，必须先与开发者确认。
- 如果现有代码不符合规范，但不影响当前任务，应记录问题，不主动扩大修改范围。

## 触发条件

- 当前需求必须修改的代码已经明显过厚，直接改动会增加风险。
- 当前需求需要复用的逻辑已经在多个位置重复出现。
- 当前代码的命名、目录或边界会妨碍理解当前改动。
- 当前问题来自既有结构缺陷，例如业务逻辑散落在模板附近、状态分散或副作用不可控。

## 重构优先级

- 优先消除纯透传 wrapper。
- 优先消除伪 hooks 聚合文件。
- 优先消除以 UI 容器词命名的业务目录。
- 优先处理页面层过厚的问题。
- 优先处理业务逻辑散落在模板附近、难以复用的问题。

## 验证要求

- 重构前应尽量补充或确认已有测试覆盖关键行为。
- 重构后必须运行受影响范围内的测试。
- 不允许通过删除、跳过或削弱测试来完成重构。

# 业务开发代码规范

## 表单

- 表单页优先拆成页面编排层、表单模型和校验 hooks、子业务区块组件。
- 表单数据应内聚在表单组件内部，父组件只负责打开、关闭、提交结果编排等页面级职责。
- 表单校验规则应靠近表单模型或专属 hook，不要散落在模板附近。
- 复杂字段映射、提交前数据装配、回显转换应拆到 hook 或 `utils/`。

## 表格

- 表格页优先拆出查询条件状态、表格列定义、列表加载与分页逻辑、页面级跳转和操作编排。
- 表格列定义过长时，不要长期堆在路由页面中。
- 行操作涉及权限、状态机或二次确认时，应抽成明确的业务操作编排。

## 弹窗与抽屉

- 多个弹窗或抽屉职责独立时，应拆成独立组件。
- 不要在单页面中用大量互不相关的布尔值统一控制所有弹窗、抽屉、表单状态。
- 弹窗或抽屉内部包含独立加载、提交、校验、回显逻辑时，优先下沉为组件和 hook。

# shared 整体设计

## 背景

项目之间经常重复维护类型、常量、正则表达式和轻量校验函数。这类能力依赖少、复用率高，适合收敛到一个稳定的基础共享包中统一维护。

## 设计目标

- 提供稳定的基础类型和常量导出。
- 保持零业务依赖、低耦合。
- 尽量使用纯函数，方便测试和复用。
- 为上层业务 Hook、组件和接口模块提供统一基础模型。

## 非目标

- 不封装接口请求。
- 不管理应用级状态。
- 不耦合 Vue 组件或 UI 组件库。

## 模块职责

### `types/`

放置通用数据模型定义，例如分页请求和分页响应。

### `constants/`

放置跨项目复用的常量和选项字典，例如证件类型、电话区号、市场枚举。

### `pattern.ts`

集中维护常用正则表达式，避免各业务项目重复定义。

### `utils/validators.ts`

放置基于常量和正则的轻量校验工具。

## 设计边界

适合放在这里的能力：

- 纯类型。
- 通用枚举和选项数组。
- 纯函数校验器。
- 稳定的正则表达式常量。

不适合放在这里的能力：

- 业务专有字典。
- 依赖页面上下文的逻辑。
- 依赖远程服务或 store 的逻辑。

## 分阶段落地建议

### 第一阶段

- 收敛重复出现的类型和字典常量。
- 统一项目内常用校验函数和正则常量。

### 第二阶段

- 为新增基础能力补充测试和 API 文档。
- 清理业务项目中重复拷贝的常量和工具实现。
